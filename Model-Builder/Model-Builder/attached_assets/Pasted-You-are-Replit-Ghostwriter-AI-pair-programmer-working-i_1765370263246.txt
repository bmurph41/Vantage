TABLE: oms

- status: enum or text (`'draft' | 'published' | 'archived'`).
- version: integer (default 1).
- settings: JSON (nullable) – holds branding, fonts, colors, page size, etc.
- created_by: FK to users.id (if available).
- updated_by: FK to users.id (if available).

TABLE: om_pages
- id: UUID.
- om_id: FK → oms.id (NOT NULL).
- title: string, e.g., 'Executive Summary', 'Financial Overview'.
- order_index: integer – for ordering pages within an OM.
- layout: JSON – stores page-level layout metadata (e.g., header/footer options, columns, background image, etc.).
- created_at, updated_at.

TABLE: om_blocks
- id: UUID.
- page_id: FK → om_pages.id (NOT NULL).
- type: string/enum: `'text' | 'chart' | 'table' | 'kpi' | 'image' | 'map' | 'custom'`.
- order_index: integer (or optionally x,y,w,h if we later use a grid layout – for now, order_index is enough).
- content: JSON – type-specific content:
  - For text: `{ markdown: string, aiGenerated?: boolean }`.

  - For chart: `{ chartType: 'bar'|'line'|'donut'|'area', config: any }`.
  - For table: `{ columns: [...], layoutOptions: ... }`.
  - For image: `{ url: string, alt?: string }`.
  - For map: `{ center: {lat, lng}, zoom, layers?: ... }`.
- data_binding: JSON – describes how the block pulls live data:
  - `source_type`: `'underwriting' | 'sales_comps' | 'rent_comps' | 'market' | 'demographics' | 'manual'`.
  - `source_id`: string | null (e.g., project ID, comp set ID).
  - `binding_role`: string key describing *what* this block wants:
    - Examples: `'om.summary.noiByYear'`, `'om.summary.revenueMix'`, `'om.salesComps.top10ByCapRate'`.
  - `query`: free-form JSON for more detailed binding (e.g., filters, timeframe, metrics).
- style: JSON – block-level styling: fonts, card style, alignment, etc.
- ai_metadata: JSON – tracks prompts, AI provider version, last generation time, etc. (for future).
- created_at, updated_at.

TABLE: om_templates
- id: UUID.
- owner_type: `'global' | 'org' | 'user'` – defines who can see/use the template.
- owner_id: nullable FK to org or user depending on owner_type.
- name: string – template name.
- scope: `'block' | 'page' | 'om'`.
- category: string – e.g., 'Executive Summary', 'Financials', 'Market'.
- template_data: JSON – a serialized description of:
  - For `scope='block'`: a single block definition.
  - For `scope='page'`: page metadata + blocks (abstracted).
  - For `scope='om'`: full OM tree (pages + blocks).
- created_by, created_at, updated_at, etc.
