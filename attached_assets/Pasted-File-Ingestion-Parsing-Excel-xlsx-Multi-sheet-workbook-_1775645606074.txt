File Ingestion & Parsing
Excel (.xlsx)

Multi-sheet workbook reading via pandas (pd.read_excel, sheet_name=None to get all sheets as a dict)
Raw header-less parsing (header=None) to handle non-standard broker/accounting layouts
Handles merged cells, sparse data, irregular row structures — reads everything as-is and lets you clean programmatically
Can target specific sheets by name or index

What this means for Vantage/StayMate: Any uploaded financial document — P&L, rent roll, operating statement, lease abstract — can be ingested and parsed into structured data, even if it's messy broker formatting.

Data Extraction & Normalization

Pattern-matched row extraction from irregular layouts (e.g., finding "GROSS SALES", "COST OF SALES", specific expense line items by label)
Cross-year reconciliation — aligned four differently-structured P&Ls into a single comparable data model
Add-back identification and categorization (owner compensation, depreciation, personal expenses, EBITDA adjustments)
Anomaly detection from raw numbers (e.g., flagging the $22K → $37 licenses drop, the 27.8% payroll tax rate, the COGS reclassification across years)

What this means for Vantage: This is essentially the P&L Parser v2 logic extended — geometry-free, label-based extraction that can handle real-world broker and accountant documents without assuming clean column structure.

Financial Modeling & Output Generation
Excel model construction via openpyxl:

Multi-sheet workbooks with distinct analytical layers
Full color-coding system (NAVY headers, LTBLUE data, YELLOW flags, GREEN positives, RED risks)
Excel formula writing (not hardcoded Python values) — =AVERAGE(B3:E3), =D5/D3, etc.
Number formatting: currency #,##0;(#,##0);"-", percentages 0.0%, multiples 0.00"x"
Dynamic row referencing for percentage calculations tied to revenue rows
Merged cells, column widths, row heights, borders
Formula recalculation via LibreOffice headless (scripts/recalc.py) with zero-error verification

Analytical outputs produced:

4-year side-by-side P&L with YoY growth, margin rows, add-back detail
Buyer normalization layer with deduction-by-deduction adjustments
Valuation scenario table (5 scenarios, color-coded by viability)
Red flag registry embedded in the model
Deal structure recommendation table
